## Ranking for adding a "workflow orchestration" capability (MCP + headless JSON + skills)

**Tier 1 — all three, plus native subagents (drop-in targets):**
1. **opencode** — best: open source, full trifecta, native subagent modes, markdown commands, HTTP server (`opencode serve`) + JS SDK, `--format json` event stream, session continue/fork from CLI.
2. **Gemini CLI** — open source, trifecta complete (`stream-json` JSONL is the richest documented event stream), md-frontmatter subagents, TOML commands; only friction: AGENTS.md needs `contextFileName` config.
3. **Amp** — trifecta + first-class TS SDK with streaming input/output, explicit `effort`/mode knobs, auto-parallel subagents with `parent_tool_use_id` threading; caveat: SDK/CLI headless consumes paid credits only.
4. **Cursor (cursor-agent)** — trifecta + subagents; NDJSON stream well-specified; closed source, account-bound.
5. **GitHub Copilot CLI** — trifecta + configurable subagents and per-agent reasoning effort; JSONL output is newer/less documented than Cursor/Gemini streams.

**Tier 2 — MCP + skills but headless is text-only:**
6. **Crush** — everything except machine-readable output (plain text from `crush run`) and no subagents yet.
7. **Kiro** — strong config surface (agents JSON w/ hooks, steering, skills) but headless requires paid API key and has no JSON output.

**Tier 3 — missing headless and/or skills:**
8. **Windsurf/Devin Desktop** — rich extension surface (MCP, skills, workflows, AGENTS.md) but zero headless CLI; orchestrate only via IDE or separate Devin cloud API.
9. **Trae** — MCP + rules in IDE but no skills/headless; the OSS trae-agent is headless-ish (trajectory JSON files) but is a separate research CLI, not the IDE agent.
10. **Aider** — no MCP, no skills, no JSON; scripting via `--message`/Python API only; project largely dormant (last release v0.86.1).

## Per-host notes

### 1. Gemini CLI (google-gemini/gemini-cli, open source)
- **MCP**: `mcpServers` object in `~/.gemini/settings.json` (user) or `.gemini/settings.json` (project). Transports: stdio (`command`,`args`,`cwd`), SSE (`url`), streamable HTTP (`httpUrl`); fields `env`, `headers`, `timeout` (default 600000 ms), `trust`, `includeTools`/`excludeTools` (exclude wins); OAuth block (`clientId`, `tokenUrl`, `authProviderType`: dynamic_discovery/google_credentials/service_account_impersonation); global `mcp.allowed`/`mcp.excluded`. https://google-gemini.github.io/gemini-cli/docs/tools/mcp-server.html
- **Skills**: Agent Skills open standard. Discovery precedence: built-in → extension-bundled → user (`~/.gemini/skills/` or `~/.agents/skills/` alias) → workspace (`.gemini/skills/` or `.agents/skills/`). Names+descriptions injected into system prompt; model calls `activate_skill`. https://geminicli.com/docs/cli/skills/ , https://github.com/google-gemini/gemini-cli/blob/main/docs/cli/skills.md
- **Slash commands**: TOML files; `~/.gemini/commands/test.toml` → `/test`; subdirs namespace (`git/commit.toml` → `/git:commit`); fields `prompt` (required), `description`; `{{args}}` (shell-escaped inside `!{...}`), shell exec `!{...}` (confirmation-gated), file injection `@{...}` (multimodal, gitignore-aware). https://google-gemini.github.io/gemini-cli/docs/cli/custom-commands.html
- **AGENTS.md**: not default; `contextFileName` (string|string[]) in settings.json can add `AGENTS.md` alongside `GEMINI.md`. https://google-gemini.github.io/gemini-cli/docs/get-started/configuration.html
- **Headless**: `gemini -p "..."` or piped stdin/non-TTY. `--output-format text|json|stream-json`. `json`: single object `{response, stats{models,tools,files}, error?}`. `stream-json`: JSONL events `init`, `message`, `tool_use`, `tool_result`, `error`, `result`. Exit codes: 0 success, 1 general/API, 42 input error, 53 turn-limit. Approval: `--yolo`/`-y`, `--approval-mode`. https://geminicli.com/docs/cli/headless/ , https://google-gemini.github.io/gemini-cli/docs/cli/headless.html
- **Subagents**: GA'd ~2026-04-15. Markdown + YAML frontmatter files in `.gemini/agents/*.md` (project) or `~/.gemini/agents/*.md` (user); body = system prompt; own context window and curated tools; disable via `{"experimental":{"enableAgents":false}}`. https://developers.googleblog.com/subagents-have-arrived-in-gemini-cli/ , https://geminicli.com/docs/core/subagents/ , https://www.infoq.com/news/2026/04/subagents-gemini-cli/
- **Effort**: none; only `-m/--model`.

### 2. Cursor — cursor-agent CLI
- **Headless**: `cursor-agent -p/--print "prompt" --output-format text|json|stream-json`. `json` result: `{type:"result", subtype:"success", is_error, duration_ms, duration_api_ms, result, session_id, request_id}`. `stream-json`: one event per line — `system` (init w/ model+permissions), `user`, `assistant`, `tool_call` (started/completed, correlated IDs), `result`; `--stream-partial-output` adds char-level deltas; thinking suppressed in print mode. https://cursor.com/docs/cli/reference/output-format
- **MCP**: CLI "automatically detect[s] and respect[s] your mcp.json" — `~/.cursor/mcp.json` global, `.cursor/mcp.json` project; `/mcp` command in CLI. https://cursor.com/docs/cli/using , https://cursor.com/docs/mcp
- **Skills**: `.cursor/skills/`, `.agents/skills/`, `~/.cursor/skills/`, `~/.agents/skills/`; compat loads from `.claude/skills/`, `.codex/skills/` (both levels). Frontmatter: `name`, `description` required; optional `paths` globs, `disable-model-invocation`, `metadata`. Invoked automatically or `/skill-name`. https://cursor.com/docs/skills
- **Slash commands**: 30+ built-ins (`/model`, `/plan`, `/ask`, `/mcp`, `/plugin`, `/sandbox`, `/resume`, `/fork`...). Custom commands originally `.cursor/commands` (changelog 1.6); skills now the primary custom-command surface. https://cursor.com/docs/cli/reference/slash-commands , https://cursor.com/changelog/1-6
- **AGENTS.md**: auto-detected at project root alongside CLAUDE.md; `.cursor/rules` loaded. https://cursor.com/docs/cli/using
- **Subagents**: Cursor 2.4 "Subagents, Skills, and Image Generation" — parallel independent agents with own context windows, custom system prompts, tool scoping. https://cursor.com/changelog/2-4
- **Other**: sessions via `agent ls` / `agent resume` / `--continue`; `--worktree [name]`; `&` prefix hands off to Cloud Agent; `--mode=plan|ask`. No effort knob documented.

### 3. GitHub Copilot CLI (github/copilot-cli; latest v1.0.68, 2026-07-01)
- **Headless**: `copilot -p "PROMPT"` (runs and exits). `-s/--silent` = agent response only. `--output-format=text|json` where json "outputs JSONL: one JSON object per line". Permissions: `--allow-all` (`--yolo`), `--allow-all-tools`, `--allow-all-paths`, `--allow-all-urls`, `--allow-tool=T`/`--deny-tool=T` (comma lists, patterns), `--no-ask-user`, `--add-dir`, `--secret-env-vars`. Sessions: `-r/--resume`, `--continue`, `--session-id`. Transcripts: `--share=PATH`, `--share-gist`. Logging: `--log-level`, `--log-dir`. Env: `COPILOT_MODEL`, `COPILOT_ALLOW_ALL`, `COPILOT_HOME`, `COPILOT_GITHUB_TOKEN`/`GH_TOKEN`/`GITHUB_TOKEN`. https://docs.github.com/en/copilot/reference/copilot-cli-reference/cli-programmatic-reference , https://docs.github.com/en/copilot/reference/copilot-cli-reference/cli-command-reference , JSON feature-request lineage: https://github.com/github/copilot-cli/issues/52
- **MCP**: persistent `~/.copilot/mcp-config.json`; per-session `--additional-mcp-config=JSON|path`; OAuth incl. `client_credentials` for fully headless auth; HTTP-header entry via MCP add/edit form (1.0.66). https://docs.github.com/en/copilot/how-tos/copilot-cli/use-copilot-cli/overview , https://github.com/github/copilot-cli/blob/main/changelog.md
- **Skills/AGENTS.md**: SKILL.md files managed via `/skills`; skills invocable as slash commands mid-input, multiple per message; AGENTS.md + CLAUDE.md + copilot-instructions combine (no priority fallback), `@`-imports expanded. https://github.com/github/copilot-cli/blob/main/changelog.md , https://docs.github.com/copilot/concepts/agents/about-copilot-cli
- **Custom agents/subagents**: `.github/agents/` (also nested + `.claude/agents`); frontmatter can set model and reasoning effort (1.0.68) and `deferred-tool-loading`; `/settings` controls subagent concurrency and depth limits; subagent model/reasoning-effort/context-tier configurable. `--agent=AGENT` routes headless runs to a custom agent. Model precedence: agent > `--model` > `COPILOT_MODEL` > config > default; aliases opus/sonnet/haiku/gpt/gemini (1.0.64).
- **Effort**: reasoning-effort per agent + subagent settings — the most granular of the closed CLIs.

### 4. opencode (opencode.ai, open source)
- **Headless**: `opencode run "msg" --format json` (docs: "raw JSON events") | `default`; `-m provider/model`, `--agent`, `-s/--session`, `-c/--continue`, `--fork`, `--file`, `--auto` (auto-approve non-denied permissions), `--attach http://localhost:4096`. `opencode serve` = headless HTTP API server (`--port`, `--hostname`, `OPENCODE_SERVER_PASSWORD` basic-auth); `opencode web` adds browser UI; JS SDK exists. https://opencode.ai/docs/cli/
- **MCP**: `mcp` key in `opencode.json` (project) / `~/.config/opencode/opencode.json` (global). https://opencode.ai/docs/config/
- **Skills**: searched: `.opencode/skills/`, `.claude/skills/`, `.agents/skills/` + `~/.config/opencode/skills/`, `~/.claude/skills/`, `~/.agents/skills/`. Frontmatter `name` (regex `^[a-z0-9]+(-[a-z0-9]+)*$`, must match dir), `description`, optional `license`, `compatibility`, `metadata`. Loaded by native `skill` tool: `skill({name})`; permission rules (allow/deny/ask, wildcards) in `opencode.json`, overridable per agent. Explicit Claude Code compatibility. https://opencode.ai/docs/skills/
- **Commands**: md files in `.opencode/commands/` and `~/.config/opencode/commands/` (or `command` key in config); frontmatter `description`, `agent`, `model`, `subtask` (forces subagent run); placeholders `$ARGUMENTS`, `$1..$n`, `` !`cmd` `` shell injection, `@file`. https://opencode.ai/docs/commands/
- **AGENTS.md**: native — project AGENTS.md (generated by `/init`), global `~/.config/opencode/AGENTS.md`; AGENTS.md preferred over CLAUDE.md when both exist. https://opencode.ai/docs/rules/
- **Subagents**: agents in `opencode.json` `agent` key or md in `.opencode/agents/` / `~/.config/opencode/agents/`; frontmatter `mode: primary|subagent|all`, `description` (required), `model`, `temperature`, `permission`, `prompt`; invoke via `@name` or automatic task tool; built-ins: build, plan (primary), general, explore, scout (subagents); child-session navigation keybinds. https://opencode.ai/docs/agents/
- **Effort**: provider-passthrough options per agent, e.g. `"reasoningEffort": "high"`. https://opencode.ai/docs/agents/

### 5. Amp (Sourcegraph/ampcode)
- **Headless**: `amp -x/--execute "prompt"`; `--stream-json` (requires `-x`), `--stream-json-thinking`, `--stream-json-input` (streaming stdin); stream-json fully threads subagents via `parent_tool_use_id` (null = main agent); completion waits for all subagents. Threads: `amp threads new|continue|fork|compact`. Env: `AMP_API_KEY`, `AMP_SETTINGS_FILE`, `AMP_TOOLBOX`. https://ampcode.com/manual/appendix , https://github.com/sourcegraph/amp-examples-and-guides/blob/main/guides/cli/README.md
- **SDK**: `npm install @ampcode/sdk`; `execute()` async-iterates messages `system|assistant|result`; options: `cwd`, `mode` (`deep|smart|rush|large`), `effort` (`none`→`max`), `continue` (thread id/true), `labels`, `visibility`, `skills` (dir path), `settingsFile`, `permissions`, `mcpConfig`, `signal`. SDK consumes paid credits only. https://ampcode.app/manual/sdk
- **MCP**: precedence `--mcp-config` flag > `amp.mcpServers` in settings.json > skill-bundled `mcp.json`; local (`command/args/env/includeTools`) and remote (`url/headers` with `${VAR}`); `amp.mcpPermissions` (matches + allow/reject); workspace servers need `amp mcp approve <name>`. https://ampcode.com/manual
- **Skills**: dirs (precedence): `~/.config/agents/skills/`, `~/.agents/skills/`, `~/.config/amp/skills/`, `.agents/skills/`, `.claude/skills/`, `~/.claude/skills/`; skills may bundle MCP servers; `amp.skills.disableClaudeCodeSkills` opt-out. Custom executable tools via toolboxes (`AMP_TOOLBOX` dirs, default `~/.config/amp/tools`). https://ampcode.com/manual
- **AGENTS.md**: cwd + parents to `$HOME`, subtree-scoped AGENTS.md, `~/.config/amp/AGENTS.md`, `~/.config/AGENTS.md`, `/etc/ampcode/AGENTS.md` (per-OS); fallback AGENT.md/CLAUDE.md; supports `@`-mentions and YAML `globs` frontmatter. https://ampcode.com/manual
- **Subagents/oracle**: automatic parallel subagents (smart mode), isolated contexts, summary-only returns; `oracle` second-opinion tool (currently Claude Fable 5, reasoning high). https://ampcode.com/manual
- **Slash commands**: none native; plugins (`amp.registerCommand`, `amp.registerTool`, `amp.on('tool.call')` permission handlers).

### 6. Crush (charmbracelet/crush, open source)
- **Config**: `.crush.json` > `crush.json` > `~/.config/crush/crush.json`; sections: models, providers, mcp, lsp, hooks, options, permissions, tools. Warning: `$(...)` in crush.json executes at load. https://github.com/charmbracelet/crush
- **MCP**: `mcp` key; types stdio/http/sse; `command/args/env`, `url/headers`; `$VAR` and `$(cmd)` expansion; `timeout`, `disabled`, `disabled_tools`. https://github.com/charmbracelet/crush
- **Skills**: `$CRUSH_SKILLS_DIR`, `~/.config/agents/skills/`, `~/.config/crush/skills/` (global); `.agents/skills/`, `.crush/skills/`, `.claude/skills/` (project); `options.skills_paths`; frontmatter `user-invocable: true` (slash-like) and `disable-model-invocation: true`; ships built-in skills. https://github.com/charmbracelet/crush
- **Context files**: CRUSH.md, AGENTS.md, CLAUDE.md, GEMINI.md (+ `.local` variants) in cwd; global `~/.config/crush/CRUSH.md`, `~/.config/AGENTS.md`; `global_context_paths` option.
- **Headless**: `crush run "prompt"` with `-q/--quiet`, `-v`, `-m/--model`, `--small-model`; global `--yolo`, `--cwd`, `--data-dir`; stdin piping; **plain text only — no JSON output flag**. https://charmbracelet-crush.mintlify.app/cli/run
- **Subagents**: not implemented; open requests https://github.com/charmbracelet/crush/issues/431 , https://github.com/charmbracelet/crush/issues/1807
- **Hooks**: shell-exec hooks, currently PreToolUse only, `command` + regex `matcher` vs tool name. https://github.com/charmbracelet/crush/blob/main/docs/hooks/README.md
- **Effort**: none documented.

### 9. Trae (ByteDance)

## Cross-cutting observations for a portable orchestration layer
- **Skills convergence**: `.agents/skills/` + `~/.agents/skills/` is now a de-facto shared path honored by Gemini CLI, Cursor, Amp, Crush, opencode, and Windsurf/Devin; `.claude/skills/` is read by opencode, Amp, Crush, Cursor, Windsurf (opt-in), Copilot (agents dir). Writing skills once to `.agents/skills/` covers most Tier-1/2 hosts.
- **AGENTS.md is near-universal** (native everywhere except Aider and Gemini-CLI-without-config).
- **Headless JSON event streams** exist in 5 hosts with near-isomorphic shapes (system/init → assistant/message → tool_call/tool_use(+result) → result), all newline-delimited: Gemini `--output-format stream-json`, Cursor `--output-format stream-json`, Amp `--stream-json`, Copilot `--output-format=json` (JSONL), opencode `--format json`. A thin adapter per host normalizes these.
- **MCP config is the least portable surface**: 10 different file paths/keys (`mcpServers` vs `mcp` vs `amp.mcpServers`), though the server-entry schema (command/args/env | url/headers) is ~identical; several hosts accept per-invocation MCP injection (Copilot `--additional-mcp-config`, Amp `--mcp-config`/SDK `mcpConfig`), which is the cleanest hook for an orchestrator to expose its own control-plane MCP server to the host.

## UNCERTAINTIES
- Aider MCP support: several third-party pages (learn.engineering.vips.edu, redcaller.com) claim aider gained MCP client support with --mcp-server flags in 2025 releases, but the official aider.chat HISTORY through v0.86.1 contains no MCP mention and issue #4506 requesting native MCP remains open — I treated mainline aider as having NO MCP; a community fork (aider-ce) may differ.
- Copilot CLI JSONL event schema: --output-format=json confirmed in official command reference, but the exact per-line object types (tool events? deltas?) are not documented in the pages fetched; verify against `copilot --help` in the shipped version.
- Gemini CLI stream-json: documented on geminicli.com headless page, but the mirrored google-gemini.github.io page only listed text|json — stream-json may be version-gated (verify minimum version).
- Trae: the ~/.trae/mcp.json path and AGENTS.md module-scoping come from search-result summaries of docs.trae.ai (page fetch was truncated); exact config paths for Trae IDE/SOLO should be re-verified. Whether SOLO standalone exposes any programmatic interface is unknown.
- Windsurf/Devin Desktop: no headless CLI found, but Cognition may expose Cascade sessions via the Devin API; I could not confirm any bridge between Devin Desktop local agent and the cloud API.
- Kiro: no JSON output flag documented for kiro-cli chat --no-interactive; also unclear whether a --agent flag selects custom agents in headless mode (docs only show /agent swap interactively).
- Cursor: legacy .cursor/commands custom-command directory (changelog 1.6) vs skills — current docs emphasize skills as the /command surface; whether .cursor/commands still loads in mid-2026 builds is unverified.
- Crush: hooks limited to PreToolUse per docs at fetch time; the config.go schema lists a hooks section that may have grown — check the repo at implementation time.
- opencode serve: one search summary described it as an ACP nd-JSON stdin/stdout server while the CLI docs call it a headless HTTP API server — likely both exist (ACP support is separate); confirm which interface the orchestrator should target.
- Dates/versions: Copilot CLI v1.0.68 (2026-07-01) and Gemini subagents (2026-04-15) are pinned; most other feature claims reflect docs as fetched 2026-07-03/04 without explicit version numbers.
